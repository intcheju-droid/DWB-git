# 더불어 함께 — 사이트 운영 안내

이 폴더 하나가 우리 매체 사이트 전부입니다. 업체 없이, 무료로, 우리가 직접 송출합니다.

## 폴더 구성

- `articles/` — **기사 원본.** 기사 1건 = 파일 1개(.md). 여기에 파일을 추가하면 발행됩니다.
- `assets/` — 디자인(style.css)과 기능(site.js). 손댈 일 거의 없음.
- `build.js` — 기사를 읽어 사이트를 만드는 프로그램.
- `.github/workflows/deploy.yml` — GitHub에 올리면 자동으로 빌드·배포하는 설정.
- `.gitignore` — 저장소에 올리지 않을 것들의 목록.

※ `public/` 폴더는 저장소에 없습니다. 빌드 결과물은 GitHub이 배포할 때마다 새로 만들며,
   컴퓨터에서 미리 보기(`node build.js`)를 하면 내 컴퓨터에만 생깁니다. 올리지 않습니다.

## 저장소 경계 규칙 — 가장 중요한 약속

이 저장소는 **Public(전체 공개)** 입니다. 그리고 GitHub은 한 번 올라간 파일을
삭제해도 커밋 이력에 영구히 남깁니다. 그러므로:

- **이미 세상에 내놓은 것만 올립니다.** 발행 완료된 기사, 사이트 코드, 운영 안내가 전부입니다.
- 취재자료, 기사 초안, 동의서, 취재원 연락처, 가명 매핑표, 내부 운영문서는
  **어떤 이유로도 올리지 않습니다.** (사무실 PC의 01~05 폴더는 저장소 밖에 삽니다.)
- 헷갈릴 때의 기준: "이 파일이 내일 아침 신문 1면에 실려도 되는가."

## 최초 1회: 무료 발행처(GitHub Pages) 만들기

1. github.com 가입 (무료).
2. 새 저장소(Repository) 만들기 — Public 선택.
3. "uploading an existing file" 링크를 눌러 **이 폴더 안의 모든 파일**을 끌어다 놓고 Commit.
   (`.github` 폴더가 함께 올라가야 자동 배포가 작동합니다.)
4. 저장소 Settings → Pages → Source를 **"GitHub Actions"**로 선택.
5. 1~2분 뒤 `https://내아이디.github.io/저장소이름` 주소로 사이트가 살아납니다.
6. `build.js` 맨 위 SITE.url을 실제 주소로 바꿔 다시 업로드 (RSS·사이트맵 주소에 쓰임).

도메인을 구입하면 Settings → Pages → Custom domain에 입력해 연결합니다.
(현재 이 저장소는 `https://dwb.ai.kr` 로 연결되어 운영 중입니다.)

## 평소: 새 기사 발행 (3단계)

1. **Claude에게**: "발행 패키지를 기사 파일로 만들어줘" → `articles/날짜-제목.md` 파일 생성.
   (또는 기존 기사 파일을 복사해 양식대로 직접 작성)
2. GitHub 저장소 → `articles` 폴더 → Add file → Upload files → 파일 올리고 Commit.
   이때 **Commit changes 칸에 커밋 메시지를 직접 씁니다** (아래 규칙 참조).
3. 끝. 1~2분 안에 자동으로 메인 페이지·기사 페이지·RSS가 갱신됩니다.

## 커밋 메시지 규칙 — 커밋 이력이 곧 발행 대장

"Add files via upload" 기본 문구를 그대로 두지 않고, 아래 접두어로 씁니다.
이렇게 하면 커밋 이력만 훑어도 무엇을 언제 발행·수정했는지 한눈에 보입니다.

| 접두어 | 쓰임 | 예시 |
|---|---|---|
| `기사추가:` | 새 기사 발행 | `기사추가: 체전연재 6회 편의시설` |
| `수정:` | 발행된 기사 고침 | `수정: 3회 오탈자, 표 숫자 정정` |
| `사이트:` | 디자인·기능·빌드 변경 | `사이트: 글자 크기 버튼 개선` |
| `운영:` | 안내 문서·설정 변경 | `운영: 경계 규칙 추가` |

## 기사 파일 양식 (.md)

```
---
title: 기사 제목
subtitle: 부제 (여러 줄은 " / "로 구분)
date: 2026-07-07
category: 복지
series: 연재명 (연재가 아니면 삭제)
seriesNo: 1
author: 남명우 기자
description: 검색·SNS 공유용 한 줄 요약
image: 사진파일명.jpg (assets 폴더에 넣고, 없으면 삭제)
imageAlt: 사진 대체텍스트 (필수)
imageCaption: 사진 설명
---
본문. 빈 줄로 문단 구분. **굵게** 는 별표 두 개.
```

## 컴퓨터에서 미리 보기 (선택)

Node.js가 설치돼 있으면 이 폴더에서 `node build.js` 실행 → `public/index.html`을 브라우저로 열기.
설치가 없어도 됩니다 — GitHub이 대신 빌드해 줍니다.
(미리 보기로 생긴 public/ 폴더는 .gitignore가 막아 주므로 실수로 올라가지 않습니다.)
