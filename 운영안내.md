# 더불어 함께 — 사이트 운영 안내

이 폴더 하나가 우리 매체 사이트 전부입니다. 업체 없이, 무료로, 우리가 직접 송출합니다.

## 폴더 구성

- `articles/` — **기사 원본.** 기사 1건 = 파일 1개(.md). 여기에 파일을 추가하면 발행됩니다.
- `assets/` — 디자인(style.css)과 기능(site.js). 손댈 일 거의 없음.
- `build.js` — 기사를 읽어 사이트를 만드는 프로그램.
- `.github/workflows/deploy.yml` — GitHub에 올리면 자동으로 빌드·배포하는 설정.
- `public/` — 빌드 결과물(자동 생성). 직접 수정하지 않음.

## 최초 1회: 무료 발행처(GitHub Pages) 만들기

1. github.com 가입 (무료).
2. 새 저장소(Repository) 만들기 — 이름 예: `dulbeo-hamkke`, Public 선택.
3. "uploading an existing file" 링크를 눌러 **이 폴더 안의 모든 파일**을 끌어다 놓고 Commit.
   (`.github` 폴더가 함께 올라가야 자동 배포가 작동합니다.)
4. 저장소 Settings → Pages → Source를 **"GitHub Actions"**로 선택.
5. 1~2분 뒤 `https://내아이디.github.io/dulbeo-hamkke` 주소로 사이트가 살아납니다.
6. `build.js` 맨 위 SITE.url을 이 실제 주소로 바꿔 다시 업로드 (RSS·사이트맵 주소에 쓰임).

도메인(예: dulbeohamkke.kr)을 구입하면 Settings → Pages → Custom domain에 입력해 연결합니다.

## 평소: 새 기사 발행 (3단계)

1. **Claude에게**: "발행 패키지를 기사 파일로 만들어줘" → `articles/날짜-제목.md` 파일 생성.
   (또는 `articles/2026-07-jangaein-cheojeon-01.md`를 복사해 양식대로 직접 작성)
2. GitHub 저장소 → `articles` 폴더 → Add file → Upload files → 파일 올리고 Commit.
3. 끝. 1~2분 안에 자동으로 메인 페이지·기사 페이지·RSS가 갱신됩니다.

## 기사 파일 양식 (.md)

```
---
title: 기사 제목
subtitle: 부제 (여러 줄은 " / "로 구분)
date: 2026-07-07
category: 복지
series: 연재명 (연재가 아니면 삭제)
seriesNo: 1
author: 남명우 기자
description: 검색·SNS 공유용 한 줄 요약
image: 사진파일명.jpg (assets 폴더에 넣고, 없으면 삭제)
imageAlt: 사진 대체텍스트 (필수)
imageCaption: 사진 설명
---
본문. 빈 줄로 문단 구분. **굵게** 는 별표 두 개.
```

## 컴퓨터에서 미리 보기 (선택)

Node.js가 설치돼 있으면 이 폴더에서 `node build.js` 실행 → `public/index.html`을 브라우저로 열기.
설치가 없어도 됩니다 — GitHub이 대신 빌드해 줍니다.

## 자립 점검표

- 콘텐츠 원본: 이 폴더(내 컴퓨터) + GitHub 저장소에 이중 보관 ✔
- 호스팅 비용: 0원 (GitHub Pages) ✔
- 업체 의존: 없음 — 원본이 전부 우리 손에 있어 언제든 다른 호스팅으로 이사 가능 ✔
- 남은 결정: 도메인 구입(연 1~2만 원, 선택), 인터넷신문 등록 표기(푸터의 괄호 채우기 — build.js 상단 SITE 설정)
